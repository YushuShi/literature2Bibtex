.gitignore 更新：
- 新增 node_modules/ 至 .gitignore，从仓库中移除已跟踪的 node_modules 目录

引用验证（citationValidator.js）：

▸ 新增 A-parallel 并行验证阶段
  CrossRef Strategy A 成功后，并行向 NCBI + Scopus 发起确认查询：
  - NCBI：以 DOI 或标题搜索 esearch，再通过 esummary 获取作者列表；
    若找到同篇文章，用 NCBI 作者替换 CrossRef 作者（NCBI 顺序更可靠，修正 CrossRef 作者顺序问题）
  - Scopus：以 DOI 确认，仅追加 badge，不取作者数据
  - 任一失败不影响主流程；两者均通过才同时显示对应 badge

▸ Bug #19 / #24 修复 — CollectiveName 过滤
  - NCBI esummary 作者解析三处均加 `.filter(a => a.authtype === 'Author')`
    防止委员会名（如 "ESE Committee"）被当作真实作者排在列表首位
  - CrossRef 作者解析三处均加 `.filter(a => a.given || a.family)`
    过滤无人名字段的机构/工作组条目（Bug #20 部分修复）

▸ Bug #24 修复 — 数据库返回集体名作为作者时跳过比较
  - 新增 `isCollective()` 函数，检测 group / committee / members / task force /
    collaboration / consortium / investigators / association 等关键词
  - 若 ref 侧全部为集体名（allRefCollective = true），跳过整个作者比较块

▸ Bug #25 修复 — 机构名作为 gen[0] 导致作者错位误判
  - gen[0] 为集体名且 ref[0] 不是时，设 genOffset = 1，跳过 gen[0] 进行作者比对
    （场景：LLM 将机构名列为第一作者，实际文章首位为真实人名）

▸ Bug #26 修复 — "et al." 字面量触发 last author 误判
  - last author check 条件从 `refAuthors.length >= 4` 改为 `genAuthors.length === refAuthors.length`
    仅在 LLM 提取的作者数与数据库完全相同（完整列表）时才比较最后一位作者
    引用截断为 "et al." 时，gen 与 ref 数量不等，自动跳过

▸ Author count check 优化
  - 触发条件从 `gen.length > ref.length` 改为 `ref.length > 1 && gen.length > ref.length + 1`
  - 允许数据库少报 1 位作者的数据质量缺口（Scopus dc:creator 常只返回首位作者）

▸ Bug #15 修复 — 期刊 ISO 4 缩写误判（journalAbbrevMatch 整合）
  - journal 比较新增 `journalAbbrevMatch()` 调用，作为 titleSimilarity < 0.7 后的第二道防线
    之前仅有 titleSimilarity 兜底，缩写形式仍可能误判

▸ Bug #17 / #23 修复 — 页码比较增强
  - 新增 `isArticleId(ep) || isArticleId(tp)` 短路：任一侧为 ≥ 6 位纯数字则跳过页码比较
    （OA 期刊如 Frontiers 用文章 ID 替代传统页码）
  - `hasRangeSep()` 扩展支持 `~` 分隔符
  - 对称修复 Bug #23：ep 含范围但 DB 只有起始页时，仅比较起始页（不误报幻觉）

▸ B-check 逻辑重构
  旧：仅当年份差 > 2 且第一作者不匹配时才触发 deep search
  新：三种独立触发条件：
    1. authorMismatch + journalMismatch → 不同同名文章，触发 deep search
    2. authorMismatch + yearMismatch    → 大概率找错文章，触发 deep search
    3. 仅 yearMismatch（作者/期刊吻合）→ 尝试 deep search 找年份更准确版本；
       找不到则保留 trueData，进入 Compare Phase 正常标记年份差异

▸ 新增 unverifiable 状态
  - invalid 后检查 BibTeX 类型：`@book|@misc|@techreport|@manual|@inbook|@incollection` 等
    非期刊类型，或无 extracted_journal / doi / pmid 的条目改为 unverifiable 状态
    （灰色显示，与 outdated 状态类似；避免对无法通过数据库验证的文献类型报 invalid）

引用验证（citationValidator.js）：

▸ Bug #27 修复 — Nature 系列文章编号 vs 印刷页码误判
  - `isArticleId()` 新增 `/^(\d{3,})-\1$/` 检测：
    CrossRef 将 Springer Nature 文章编号提交为 "N-N"（首尾相同的伪 range），如 "1099-1099"
  - CrossRef 仲裁阶段：`isArticleId(crPg)` 为真时直接 resolved = true，不与印刷页码比较

▸ Bug #28 修复 — 预印本标题 vs 发表标题不一致导致 Not Found
  - CrossRef B-search 新增低置信档（0.70 ≤ sim < 0.85）：
    设 `lowConfidenceTitle = true`，照常走 Compare Phase
  - Compare Phase 结束后统计 mismatch 字段类别数（author / year / journal / volume / pages 各一类）：
    ≥ 2 类 → 强制 status = invalid（很可能是不同文章），清空 trueData / correctedBibtex / sources
    ≤ 1 类 → 正常通过或标记 corrected
  - Scopus B-search 阈值保持 0.85 不变

▸ Bug #29 修复 — 作者不匹配无法区分顺序问题与捏造
  - 新增 `crossRefAuthors` 变量：Strategy A 阶段在 NCBI A-parallel 可能替换作者前保存 CrossRef 作者列表
  - Strategy B CrossRef 仲裁阶段同步收集 CrossRef 作者（零额外 API 请求）
  - Compare Phase 作者不匹配时检查 gen 作者是否出现在 NCBI + CrossRef 合并列表的任意位置：
    找到 → 提示 "found in different position"（顺序差异，非捏造）
    找不到 → 提示 "possibly fabricated"（高度可疑的幻觉）

▸ NCBI DOI 提取修复
  - esummary DOI 提取从 `r.elocationid?.replace('doi: ', '')` 改为 `r.articleids.find(a => a.idtype === 'doi')?.value`
    NCBI 返回格式不一致，elocationid 有时为空；articleids 数组更可靠
  - 若 articleids 中仍无 DOI，fallback 调用 NCBI ID Converter API
    (`/pmc/utils/idconv/v1.0/?ids=<pmid>&format=json`)，将 PMID 转换为 DOI
  - 修复于 Strategy A esummary + Strategy B NCBI 路径两处

▸ 新增 CrossRef 仲裁阶段（Arbitration Phase）
  - Compare Phase 将 journal / year / volume / pages 差异暂存至 `pendingMismatches`（而非立即标 hallucination）
  - 若 trueData 来源非 CrossRef 且有 DOI，额外调用 CrossRef works API 进行仲裁：
    - pages：比较提取值与 CrossRef page 字段；CrossRef 为 isArticleId 时直接 resolved
    - journal：journalAbbrevMatch + titleSimilarity ≥ 0.7
    - year：差 ≤ 1 为 resolved
    - volume：normalize 后相等或包含关系
  - resolved → 不标 hallucination（可能是 NCBI/Scopus 与 CrossRef 的元数据版本差异）
  - unresolved → mismatch 提交，isHallucinated = true
  - CrossRef 仲裁成功时将 "CrossRef" 追加至 sources 列表
  - authors 不参与仲裁（NCBI 作者数据质量优先，直接提交）

▸ journalAbbrevMatch 副标题剥离
  - 新增 `stripSubtitle()`：将 NCBI 格式副标题（" : official journal of the Society of XXX"）在 " : " 处截断
  - 使用 `\s+:\s+` 匹配（带空格），避免误切 "CA: A Cancer Journal for Clinicians" 中的冒号

▸ 新增 stripEloc() 辅助函数
  - 剥离页码字段末尾的电子定位符号（electronic location indicator）
    示例："949-963.e18" → "949-963"，"1653–1666 e7" → "1653–1666"
  - 要求主体为纯数字范围，防止将独立 article ID（如 "e15"）误剥离
  - 应用于 Compare Phase 页码比较的 ep / tp 提取
