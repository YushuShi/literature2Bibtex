2026-03-12 | feat: multi-format output, download, arXiv validation, PDF paste support
Commit: d0f8c47
--------------------------------------------------------------------------------

▸ 新功能：PDF / 纯文本引用列表直接粘贴解析（splitCitations）
  - bibtexConverter.js 新增 `splitCitations()` 函数，自动识别多种格式拆分引用条目：
    - Strategy 1a：`[N]` / `(N)` 括号编号（限 ≤ 3 位数字，排除 4 位年份）
    - Strategy 1b：`N.` 格式（要求点后空格 + 大写字母，排除 DOI 中的点）
    - Strategy 2：空行分隔段落
    - Strategy 3：每行一条（兜底）
  - 超过 16 条自动分批（CHUNK_SIZE=16），调用 LLM 时分批发送，支持大规模引用列表
  - 分批模式检测截断（received < sent）并设 truncated=true

▸ 新功能：多格式引用输出（9 种学术格式）
  - 新建 src/utils/formatCitation.js，支持：
    APA / MLA / Chicago / Vancouver / IEEE / AMA / AMJ / ACS / Harvard Gatton
  - 界面新增格式切换按钮：BibTeX / APA / MLA / IEEE（主要）+ Other 下拉菜单（其余格式）

▸ 新功能：引用下载（3 种格式）
  - 右上角新增下载按钮 + 下拉菜单：
    BibTeX (.bib) / RIS (.ris) / NBIB (.nbib)
  - buildRIS() / buildNBIB() 内联于 formatCitation.js

▸ 新功能：arXiv 预印本验证与 BibTeX 生成
  - Strategy A（Cite.async）能直接解析 arXiv DOI（`10.48550/arXiv.XXXX`）
  - 生成标准 @misc BibTeX 条目，包含 eprint / archivePrefix / primaryClass 字段
  - 修复 CrossRef CORS 错误：移除被禁止的 User-Agent 请求头

▸ 新功能：Semantic Scholar 作为第四路验证源
  - Strategy B 标题搜索新增 Semantic Scholar API（原生支持 CORS，无需代理）
  - 覆盖 arXiv / 预印本 / 计算机科学领域文献
  - B-search 结果 Semantic Scholar 分配紫色 badge

▸ UX：截断警告 banner
  - LLM 批量返回条数少于发送条数时，结果页顶部显示黄色警告 banner
  - 提示用户检查 API key 输出 token 限制或代理设置

Commits: ad6aaff / ace1124 / 822f555

▸ 新功能：Strategy C — 预印本 ID 主动扫描
  - Strategy A/B 均失败后，从原始引用文本扫描预印本 ID：
    - arXiv：匹配 arXiv:XXXX.XXXXX，转换为 DOI `10.48550/arXiv.XXXX`
    - bioRxiv / medRxiv：匹配 doi.org/10.1101 路径
    - SSRN：匹配 ssrn.com/abstract=XXXXXX 或 SSRN=XXXXXX
  - 找到预印本 ID 后走 Cite.async 解析，生成 @misc BibTeX

▸ 新功能：标题精搜（deepSearch.js）
  - 新建 src/utils/deepSearch.js：B-check 检测到可能找错同名文章时调用
  - 以 CrossRef + Semantic Scholar 两路带年份过滤重新搜索
  - 对候选结果打分（年份精确 +3，年份差1 +1，首位作者匹配 +2），score ≥ 2 才采用
  - 通过 Cite.async 解析 DOI 生成完整 BibTeX；无 DOI 时手工拼接

▸ 验证鲁棒性改进：
  - 年份比较：差 ≤ 1 不标幻觉（online-first vs print 出版年差异）
  - BibTeX 输出：有 correctedBibtex 时始终采用数据库版本（即使 status=valid）
